jobs - список выполняемых фоновых задач
kill %2 - отправить сигнал SIGTERM второй фоновой задаче
<C-z> - приостановление текущей задачи

Операторы перенаправления ввода-вывода:

cmd1 > file1
перенаправление вывода cmd1 в созданный file1; если такой уже существует, он
будет перезаписан с нуля

cmd1 >> file1
то же, но если file1 уже существует, вывод будет дописан в конец файла

cmd2 < file2
запуск cmd2 с подачей содержимого file2 на ее стандартный ввод 

cmd3 > file1 < file2
перенаправление и ввода, и вывода

cmd1 | cmd2
запуск cmd1 и cmd2 с подачей стандартного вывода cmd1 на стандартный ввод
cmd2 (конвейер)

cmd4 2> errfile
поток сообщений об ошибках в errfile

cmd5 2>&1 | cmd6
поток стандартного вывода и диагностики cmd5 в стандартный ввод cmd6

<alt+.> - повтор предыдущей команды

Команды vim
^ - начало строки
$ - конец строки
U - отмена всех изменений, внесенных в текущую строку
J - удаление перевода строки
. - повтор последней операции

:e <name> - начать редактирование еще одного файла

<C-^> - переключение между двумя последними редактируемыми файлами (*)
(*) чтобы редактор не требовал сохранения текущего файла для переключения,
    следует задать эту настройку командой :set hidden

Права доступа к файлам

С каждым файлом в unix связано 12-битное слово - permissions, например:
    $ ls -l /bin/cat
    -rwxr-xr-x 1 root root 35280 Feb  7  2022 /bin/cat
здесь первый знак показывает тип файла ("-" - file "d" - directory "l" - link)
далее - права доступа для владельца, группы, всех остальных соответственно.

Права доступа обозначают восьмеричным числом (например, для cat это 0755),
при этом порядок цифр такой же, как в выдаче ls -l
